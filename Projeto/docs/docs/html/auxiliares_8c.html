◆ aplicaRegras()

bool aplicaRegras ( ESTADO *  e)

Aplica um conjunto de regras lógicas sobre o tabuleiro.

Parameters
ePonteiro para a estrutura de estado do jogo.
Returns
true se pelo menos uma modificação foi feita no tabuleiro.
false se nenhuma alteração foi realizada.

Esta função implementa três regras fundamentais de inferência:

  1. riscarLetrasIguaisBrancas():
    • Se uma letra está em branco (maiúscula), todas outras instâncias da mesma letra na mesma linha/coluna devem ser riscadas.
  2. pintarVizinhasDeRiscadas():
    • Toda a célula riscada (#) deve ter suas 4 vizinhas ortogonais convertidas para branco (maiúsculas).
  3. pintarCasasQueIsolamLetras():
    • Se riscar uma célula criaria áreas isoladas de letras, converte essa célula para branco em vez de riscar.

Parameters

Código de status:
  • E_OK: Entrada válida processada.
  • E_EOF: Fim de entrada
  • E_VAZIO: Linha vazia ou apenas espaços.
  • E_FORMATO_ARG: Formato de comando inválido.

Regras de formatação:

  • Comandos sem argumentos: "c"
  • Comandos com argumentos: "c argumento"
  • Não são permitidos:
    • ra1
    • r a1 2

bool resolverPuzzle ( ESTADO *  e,
ESTADO *  original 
)

Implementa a lógica de resolver o puzzle.

Parameters
ePonteiro para o estado atual do jogo (modificado durante a resolução).
originalPonteiro para cópia do estado original (usado para restauração).
Returns
true se o puzzle foi resolvido com sucesso.
false se o puzzle for insolúvel a partir do estado atual.

Esta função implementa um algoritmo :

    • Aplica repetidamente regras
    • Verifica validade após cada alteração
    • Aborta se violar regras básicas
    • Seleciona a primeira célula minúscula
    • Testa hipótese 1: converter para branco (maiúscula)
    • Se falhar, testa hipótese 2: riscar ('#')
    • Recursão
  3. Mecanismo de retrocesso:
    • Restaura estado original após cada ramo mal-sucedido
    • Interrompe busca quando encontra primeira solução
